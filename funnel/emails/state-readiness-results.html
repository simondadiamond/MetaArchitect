Each gap below is a structural problem — not a prompt problem. Here's the architectural fix.

S

01  /  Structured

What's actually missing

Your system has no persistent state schema — when a run crashes, there's no record of where it stopped, so it restarts from scratch.

Define a typed state object with a stage field and write it to durable storage at every transition. On startup, check for an existing state record before executing — never reconstruct position from conversation history.

T

02  /  Traceable

What's actually missing

There's no structured trace tied to individual runs — you can't pull up what happened in a specific session, and you can't explain why behavior changed between runs.

Assign a shared workflow_id to every run and log every LLM call, tool call, and stage transition with it. Version your prompts and models explicitly — unversioned components make behavior changes permanently undiagnosable.

A

03  /  Auditable

What's actually missing

Automated decisions are being made with no record of what data or model version produced them — one compliance request and you're starting from nothing.

Log model version, prompt version, input data, and output for every automated decision as a separate decision record. The trace tells you what happened — the decision record tells you why. You need both.

T

04  /  Tolerant

What's actually missing

Your workflows have no resumption logic — a failure at step 8 restarts everything from step 1, re-running expensive calls and risking duplicate writes.

Check for a committed stage value on startup and resume from that point. Every external dependency needs an explicit retry policy and a defined degraded mode — undocumented fallback is the same as no fallback.

E

05  /  Explicit

What's actually missing

LLM outputs flow directly into writes and external actions — there's no deterministic gate between what the model reasons and what the system does.

Add a validation function after every LLM call that checks the output against a schema before any write or tool call executes. If validation fails, route to an error path — never silently continue with unvalidated output.
