What your score means — and where to start
0–3
Critical Risk
One incident away from a rebuild. Start with S — define your state schema before anything else moves.
4–5
High Risk
Failures are happening — you can't see them yet. Add T logging to one workflow this week. Visibility before fixes.
6–7
Developing
Survives demos. A is where regulation finds you. Write a decision record for one automated output before you ship.
8–9
Prod-Ready
Shippable. Close the last gap. Run the Reboot Test below before you call it production-ready.
10
STATE ✓
Auditable, resumable, observable. Verify quarterly. New workflows reset the score.
The Reboot Test — before you call it production-ready
1.
Restart the service mid-run. Does it resume from where it stopped?
2.
Inject a failure at step 6 of 10. Does the error path clear the lock and report cleanly?
3.
Take a dependency offline for 2 minutes. Does the system degrade gracefully or fail silently?
4.
Mock the LLM node with a worst-case output. Does it reach a real-world action, or does the validation gate catch it?
Most teams score 4–6. The workshop walks through closing each gap on a real production system — live teardown, no slides.
