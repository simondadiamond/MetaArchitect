## What it is

Simon's daily-driver agent factory, genericized: a kanban-based pipeline driving parallel Claude Code sessions with **STATE-compliant gates between every stage** (plan → code → test → visual-verify → PR → gated auto-merge). The product is the methodology + working setup + the reasoning — not a UI. Tools like Vibe Kanban and Emdash prove the demand and sell interfaces; nobody credibly sells the operating discipline.

## Contents (v1 scope)

- Kanban structure + story schema (what a well-formed story looks like, routing criteria)
- Plan / code / test / verify skill definitions with the explicit validation gates
- CLAUDE.md conventions that make agents reliable (workflow discipline, one-job-one-PR, lessons loop)
- Setup guide (fresh repo → working pipeline)
- Design rationale — why each gate exists, what failure it prevents (this is the differentiator; it's "State Beats Intelligence" applied to the dev loop)

Out of v1: video course, install-with-you session (Tier 2 — build only after the kit sells), any hosted component.

## Job description (what success means)

Proof asset + Team Training lead-gen (offer-ladder §2 tier 3) + covers the Claude subscription. NOT a revenue pillar. $1–2k lifetime + 2 workshop inquiries = success.

## Execution plan (agent-driven, Simon reviews)

1. **Extract (agents, background):** genericize the pipeline — strip Simon-specific config, secrets, repo names; write the setup guide + rationale docs. Simon-hours: review + dogfood on a fresh project, capture 2–3 before/after metrics.
2. **Tease:** build-in-public posts (Meta Layer pillar) documenting the extraction. These are ladder-funnel content that happen to build a kit waitlist. Waitlist target: 50+ by day 30 — or an honest no-demand signal.
3. **Launch:** payment page + the three native-channel posts. Checkpoint: 10 sales (~$1.5k).
4. **Compound or park (day 90):** if selling — consider Tier 2 and add the "workshops available" line prominence. If <$500 and flat — park the kit, keep the content spine (it feeds the ladder regardless).
